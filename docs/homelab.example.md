# homelab 拓扑参考(示例)

把本文件复制到 `~/.config/voidmaker/homelab.md` 并改成你自己的家庭网络/服务分布,
`homelab_topology` 工具会原样读取它,供角色回答「家里网络怎么连的 / 某服务网址是多少」。

本文件是**个人基础设施信息**(内网地址、主机名、服务清单),`~/.config/voidmaker/`
下的真实版**不入代码库**——仓库里只留这份占位示例。

同时在 `~/.config/voidmaker/config.yaml` 配置 hub 聚合层地址:

```yaml
homelab:
  enabled: true
  hub_url: http://<你的内网主机>:<端口>   # homelab-hub 的 /rk 聚合端点
```

---

下面是格式示例,按你的实际情况改写(不要提交真实版):

```
家庭 homelab 拓扑(N 台机 + 隧道):

- 主力机「<主机名>」:角色 / 内网网关 / 监控。地址 <内网 IP>。
- 媒体机「<主机名>」:媒体 / 相册 / 下载。地址 http://<主机名>:<端口>。
- VPS(公网 <IP/域名>):反代 + 告警。

服务准确访问地址(打开服务时用这张表,别凭泛域名瞎拼):
  Jellyfin 媒体库   → http://<主机>:8096
  Immich 相册       → http://<主机>:2283
  qBittorrent 下载  → http://<主机>:8080
  ...

关键约束:<例如某机在 NAT 后只能出站、数据靠中继上报>
已知隐患:<例如某盘偏满、无异地备份>
(密码库内容绝不查询、绝不读取。)
```
